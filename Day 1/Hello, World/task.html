For your first Java program, you will learn to print out something to the console. To do this, type in public class HelloWorld{}

Writing this line of code specifies the name of the file to the Java decompiler.

  • Public specifies that any other programs can see the contents of the Java file. This becomes important in large programs where many Java files are used
  • Class specifies that the file being written is in fact a Java Class file. Java Class files just mean that the file is filled with code to be run. Although other types of Java files exist, like enumerations and interfaces, those are not used as much and are for more advanced programs.
  • HelloWorld is the name of the Java Class. You can name your Java class whatever you want as long as it only has letters, but as a general rule of thumb, capitalize the first letter of every word.
  • Lastly, the curly braces specify to Java that everything within them is part of the HelloWorld Java Class.

Next type in public static void main(String[] args){} in between the curly braces. You should have public class HelloWorld{public static void main(String[] args){}}

This code is the main method or the part of the program that Java runs when the Java class is run. Let’s go through each part of it.

  • Public once again specifies that any other files can see the contents of the Java method.
  • In Java, a static member is a member of a class that isn’t associated with an instance of a class. Instead, the member belongs to the class itself. As a result, you can access the static member without first creating a class instance. This is kind of complicated and will be explained later in the course. For now, remember that the main method always has static in it.
  • Void specifies to the decompiler that this method will not return anything. This also may confuse you, but we will go over methods in more detail later in the course.
  • Main is the name of the method. Methods can be called anything as long as it only uses letters, but for Java to run the method, it must be called main. Again, you will learn more about methods later.
  • In the parenthesis, we have String[] args. A String is one of the data types which you will learn about today. It is just a string of characters combined, hence the name string. The [] part signifies that it is actually a string array, which means that it is a bunch of strings combined. If a string is like a word, imagine a string array to be a sentence, which is multiple words combined.
  • Finally, the curly braces signal that everything inside them is part of the main method, meaning they will get run.

You now have a space where you can run code and have it work, but it’s pretty hard to look at. Since things in Java are split up by curly braces and parentheses, Java doesn't really care about how long spaces are. This means you can format code to make it look better. Try pressing enter after each curly brace to make the code easier to look at. It should look like this:

   
public class HelloWorld{
   public static void main(String[] args){
       //this is where you enter code
   }
}

You can also add tabs like we did to make it easier to see what the curly braces contain. In the future, you can format your code easily by using an IntelliJ shortcut: Ctrl+Alt+L (CMD+OPTION+L for MacOS). However, not all IDE’s, or development environments have this, so don’t rely on it too much.

In the above piece of code, you might have noticed the double slashes followed by “this is where you enter code”. This is called a comment. The double slashes signify a comment. A comment is something the decompiler will skip over, meaning that the decompiler doesn’t run it. Comments are a good way to add notes to your code. Comments are often used as reminders, a tool in debugging, and also explanations of complicated code so that others can understand what each line does more easily. In Java, a single line comment can be anything followed by a double slash(//), and a multiline comment starts with /* and ends with */, with any text you want in between.

Now, it’s time to write the line of code that allows you to actually print a line to the console. Underneath the comment would, type System.out.println(“Hello, World!”) Once again, let’s go through each part of this statement.

  • System signifies that the command is calling the system. In this case, you are using the system to print something to console.
  • Out signifies that the statement is part of the output stream, meaning it is something that sends something to the user. In this case, whatever you are printing in console.
  • Println tells the decompiler to print on a new line. If you want to print on the existing line, you can just use System.out.print, but it is not used as often.
  • The parenthesis tells the decompiler to print whatever is inside. You can put a lot of things inside those parentheses like variables and operators, but to print words like Hello, World!, you need to specify to the decompiler that you are typing a string, which as mentioned above, is a grouping of characters. Again, you’ll learn more about strings today. To tell the decompiler that you are typing a string, just use quotation marks.
