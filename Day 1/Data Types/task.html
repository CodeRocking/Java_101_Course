Lab 2: Data Types and Variables

What Can You Do With Variables?

Learning to Use Different Variable Types

In this task, you will learn to work with the data types and variables. First, initialize a basic variable. Type in the following code into the main method: int number = 100; Refer to the previous lab if you need help making the main method, or ask for help. Like the previous lab, let’s go through each part of the code you wrote.

  • int specifies the data type you are assigning. Since 100 is a number, you should use int. Although you can use other data types like byte, short, and long, int is the most widely used data type to store numbers in Java.
  • Number is used as the name for the variable. This can be anything as long as it uses letters, numbers, and underscores. However, it can only start with a letter. As a rule of thumb, programmers use camelCase to make variables easier to read. This means starting off lowercase and then using uppercase to signify the starting of a new word. Make sure your variable names are descriptive.
  • = is just what Java uses to set the value of something. In Java, = is NOT used to check if two things are the same value. That task instead falls to the == operator. You will learn more about operators later today.
  • Finally, 100 is the value that we set the variable called number to. In this case, it is 100, but it can be anything within the limits of the integer data type. We follow this with a semicolon to signal the end of the command/line

A cool thing you can do with variables is print them out. Under the variable you created, type System.out.println(number)Notice that there are no quotation marks. This is because we are not writing a string but instead just feeding a variable into it. Try running the code, and check to see if the output is 100. Ask for help if this doesn’t work correctly. Next, try changing the value stored in the number from 100 to any number of your choosing. Check to see if printed number changes.

You now know how to create and print a basic variable. However, you can create many more variable types. Try making and printing a variable of each type by typing more lines of code underneath the previous. Then show us after we finish so that we can check.
